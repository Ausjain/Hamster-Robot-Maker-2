현재 프로젝트를 새로 만들지 말고 기존 프로젝트를 직접 수정해 줘.

현재 구현된 미로찾기, 라인트레이싱, 명령 실행, 충돌 판정 등의 기능은 유지하고, 다음 두 부분을 함께 개선한다.

미로찾기를 고정된 5단계 게임형 미션으로 변경
명령값을 별도 설정창이 아니라 각 명령 카드 안에서 바로 조절하도록 변경

첨부 이미지는 현재 프로젝트 화면이다. 현재 디자인과 전체 기능을 유지하되 아래 요구사항대로 수정해 줘.

2. 미로찾기를 5단계 고정 맵으로 변경

현재 쉬움·보통·어려움 또는 랜덤 맵 구조를 제거하고, 미로찾기를 다음 5개의 고정 단계로 구성한다.

상단에는 다음 버튼을 표시한다.

2단계
3단계
4단계
5단계

현재 선택한 단계는 색상과 테두리로 분명하게 강조한다.

각 단계의 맵은 미리 정해진 고정 맵으로 저장한다.

새로고침해도 같은 단계에는 같은 맵이 표시되어야 한다.
다시 시작을 눌러도 맵은 바뀌지 않는다.
단계 버튼을 누르면 벽, 출발점, 출발 방향, 목표점이 실제로 변경되어야 한다.
랜덤 미로 자동 생성 기능은 사용하지 않는다.
각 단계는 코드상 서로 다른 맵 데이터로 정의한다.

각 단계 화면에 다음과 같이 표시한다.

기본 이동과 한 번의 방향 전환

4. 단계 이동 방식
처음에는 1단계를 기본으로 표시한다.
한 단계를 성공하면 다음 단계 이동 버튼을 표시한다.
상단의 1~5단계 버튼을 눌러 교사가 직접 단계 이동도 할 수 있게 한다.
단계 이동 전에 작성한 명령이 있으면 확인창을 표시한다.

확인 문구:

단계를 바꾸면 현재 작성한 명령이 삭제됩니다. 이동할까요?

6. 명령 편집 방식 변경

현재는 내 명령 순서의 작은 블록을 클릭해야 별도의 큰 설정창이 나타난다.

이 방식을 삭제한다.

선택한 명령 하나만 별도 패널에서 편집하는 구조를 사용하지 않는다.

대신 각 명령 카드 자체에 모든 조절 기능을 항상 표시한다.

학생이 명령을 추가하면 내 명령 순서 영역에 펼쳐진 카드가 바로 추가되게 한다.

각 명령 카드는 기본적으로 펼쳐진 상태여야 한다.

7. 명령 추가 버튼

다음 네 버튼을 표시한다.

＋ 앞으로 이동
＋ 뒤로 이동
＋ 왼쪽 돌기
＋ 오른쪽 돌기

버튼을 누르면 내 명령 순서의 맨 아래에 해당 명령의 펼쳐진 편집 카드가 추가된다.

추가된 카드 위치로 자동 스크롤한다.

11. 명령 카드 배치

내 명령 순서는 작은 블록 가로 배열이 아니라 펼쳐진 편집 카드의 세로 목록으로 표시한다.

예:

내 명령 순서

[1번 앞으로 이동 카드]

[2번 오른쪽 돌기 카드]

[3번 앞으로 이동 카드]

명령이 많아지면 내 명령 순서 영역 안에서 세로 스크롤이 가능하게 한다.

카드 글씨와 버튼 크기는 줄이지 않는다.

각 카드에는 선택 여부와 상관없이 조절 기능이 계속 보여야 한다.

별도의 선택한 명령 설정 패널은 만들지 않는다.

가장 중요한 조건
기존 프로젝트를 새로 만들지 않는다.
랜덤 맵이 아니라 서로 다른 고정 5단계 맵을 사용한다.
명령 블록을 클릭해야 설정이 나타나는 방식을 제거한다.
모든 명령 카드는 기본적으로 펼쳐진 상태다.
각 카드 안에서 속도, 시간, 각도를 바로 조절한다.
라인트레이싱 기능을 삭제하지 않는다.
인쇄용 학습지 형태로 바꾸지 않는다.
초등학생이 실제 햄스터 실습 전에 게임처럼 반복 실행하고 수정하는 시뮬레이터로 유지한다.
