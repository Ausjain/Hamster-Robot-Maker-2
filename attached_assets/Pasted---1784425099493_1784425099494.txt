현재 프로젝트의 기능과 코드는 유지하고, 첨부한 현재 화면을 기준으로 사용자 인터페이스를 개선해 줘.

새 프로젝트를 만들거나 기존 기능을 단순화하지 말고 현재 화면을 수정해 줘.

현재 구현된 다음 기능은 반드시 유지한다.

미로찾기와 라인트레이싱 탭
쉬움·보통·어려움 난이도
새 미션 또는 다음 맵
앞으로·뒤로·왼쪽 돌기·오른쪽 돌기
속도, 시간, 각도 조절
각도 0.1도 단위 미세 조절
명령 순서 드래그 변경
한 단계 실행, 전체 실행, 일시정지, 다시 시작
충돌 및 도착 판정

라인트레이싱 관련 코드와 화면을 삭제하거나 숨기지 말 것.

1. 전체 화면 배치 개선

현재 명령 설정 카드가 화면의 대부분을 차지해 미로와 내 명령 순서가 너무 작게 보인다.

데스크톱 화면에서는 다음과 같이 재배치해 줘.

왼쪽 영역: 전체 너비의 약 42%
미로 화면
출발 방향
현재 위치와 방향
힌트 보기
실행 상태

미로 화면은 현재보다 약 30~40% 크게 표시해 줘.

오른쪽 영역: 전체 너비의 약 58%

위에서부터 다음 순서로 배치해 줘.

실행 버튼
내 명령 순서
명령 추가 버튼
선택한 명령의 설정 패널

학생이 가장 자주 확인하는 내 명령 순서가 명령 설정 카드보다 위에 보이게 해 줘.

2. 명령 추가와 명령 설정을 분리

현재 앞으로·뒤로·왼쪽·오른쪽 카드가 항상 큰 설정 패널로 표시되어 화면을 너무 많이 차지한다.

평소에는 다음 네 개의 간단한 추가 버튼만 한 줄로 표시해 줘.

＋ 앞으로
＋ 뒤로
＋ 왼쪽 돌기
＋ 오른쪽 돌기

버튼을 누르면 해당 명령이 내 명령 순서에 추가되게 한다.

명령을 추가하는 버튼에는 슬라이더나 세부 설정을 넣지 않는다.

학생이 내 명령 순서에 있는 명령 블록을 클릭했을 때만 아래에 선택한 명령 설정 패널을 열어 줘.

예:

선택한 명령: 오른쪽 돌기

각도 92.6°
[슬라이더]
[-10] [-1] [-0.1] [직접 입력] [+0.1] [+1] [+10]

아무 명령도 선택하지 않은 상태에서는 다음 문장을 표시해 줘.

수정할 명령 블록을 눌러 주세요.

3. 내 명령 순서 강조

내 명령 순서 영역을 지금보다 크게 만들어 줘.

각 명령 블록에는 반드시 다음 설정값이 바로 보여야 한다.

앞으로: 속도 50 · 1.5초
뒤로: 속도 40 · 0.8초
왼쪽 돌기: 87.3°
오른쪽 돌기: 92.6°

각 블록에는 다음 기능을 제공한다.

클릭하여 설정 열기
드래그하여 순서 변경
개별 삭제
현재 실행 중인 블록 강조
실행 완료된 블록 체크 표시

명령이 많아지면 영역 안에서 가로 또는 세로 스크롤이 가능하게 하되, 블록 글자가 작아지지 않게 해 줘.

현재 선택한 블록은 밝은 테두리로 분명하게 표시해 줘.

4. 각도 직접 입력 개선

왼쪽 돌기와 오른쪽 돌기의 각도는 학생이 다음 방법으로 모두 조절할 수 있게 한다.

슬라이더
숫자 직접 입력
미세 조정 버튼

각도 조건:

최소 1.0°
최대 180.0°
기본값 90.0°
소수점 첫째 자리까지 허용
입력 단위 0.1°

미세 조정 버튼:

−10°
−1°
−0.1°
+0.1°
+1°
+10°

숫자 입력창은 type="number"로 만들고 step="0.1"을 적용한다.

학생이 87.3처럼 직접 입력한 뒤 Enter를 누르거나 입력창 밖을 누르면 즉시 명령 블록과 시뮬레이션에 반영되게 한다.

입력값을 정수로 반올림하지 말고 소수점 첫째 자리까지 유지한다.

5. 속도와 시간 직접 입력

앞으로 이동과 뒤로 이동도 슬라이더만 사용하지 말고 숫자를 직접 입력할 수 있게 해 줘.

속도:

0~100
1 단위
−10, −1, +1, +10 버튼

시간:

0.1~5.0초
0.1초 단위
−1, −0.1, +0.1, +1 버튼

명령 블록의 설정값이 변경될 때 예상 이동거리 표시도 즉시 갱신해 줘.

다만 학생에게 정확한 실제 거리라고 오해되지 않도록 다음처럼 표시해 줘.

시뮬레이션 예상 이동거리: 약 150px

또는

가상 이동거리: 3칸

6. 미로 화면 정보 보강

미로 화면 안에서 햄스터가 바라보는 방향을 현재보다 더 크게 표시해 줘.

햄스터 옆에 선명한 방향 화살표를 표시하고, 미로 아래에는 다음 정보를 한 줄로 표시해 줘.

현재 위치: (60, 100) | 방향: 오른쪽 0° | 실행 단계: 0/3

벽에 충돌하면:

충돌 위치에 빨간 X 표시
해당 명령 블록을 빨간색 테두리로 강조
2번째 명령에서 벽에 부딪혔어요. 표시

목표에 도착하면:

목표 깃발 주변에 성공 효과
미션 성공! 총 5개의 명령을 사용했어요. 표시

명령이 끝났는데 도착하지 못하면:

명령은 끝났지만 목표에 도착하지 못했어요. 마지막 이동거리나 회전 각도를 조정해 보세요. 표시
7. 다시 시작 버튼 동작 구분

버튼의 역할을 명확히 구분해 줘.

다시 시작: 햄스터의 위치와 방향, 실행 단계만 초기화하고 작성한 명령은 유지
명령 모두 지우기: 작성한 명령 전체 삭제
새 미션: 미로, 출발점, 출발 방향, 목표점 변경

현재 다음 맵이라는 이름은 새 미션으로 바꿔 줘.

새 미션을 누르면 작성한 명령이 삭제될 수 있다는 확인창을 표시해 줘.

8. 힌트 기능

미로 화면 위에 힌트 보기 버튼을 추가해 줘.

난이도별 힌트:

쉬움: 출발점부터 목표까지의 대략적인 이동 경로를 흐린 점선으로 표시
보통: 다음에 필요한 동작 하나만 표시
어려움: 힌트 버튼 숨김

힌트는 완성된 정답 명령 목록을 그대로 보여 주지 않는다.

학생이 예시를 그대로 복사하지 않도록, 방향 또는 경로만 안내하고 구체적인 속도·시간·각도 값은 보여 주지 말 것.

9. 라인트레이싱 탭 유지 및 확인

라인트레이싱 탭이 화면에만 존재하고 빈 화면으로 나오지 않도록 실제 기능을 확인해 줘.

라인트레이싱 탭에는 최소한 다음 항목이 보여야 한다.

수정 완료 전에 반드시 다음 항목을 직접 테스트해 줘.

10. 아이패드 화면

아이패드 가로 화면에서는 데스크톱과 비슷하게 좌우 배치를 유지한다.

아이패드 세로 화면에서는 다음 순서로 배치한다.

미로 또는 코스
실행 버튼
내 명령 순서
명령 추가 버튼
선택한 명령 설정
실행 정보와 도움말

터치 영역은 최소 44px 이상으로 만들고, 버튼이나 숫자 입력창이 서로 너무 붙지 않게 한다.
