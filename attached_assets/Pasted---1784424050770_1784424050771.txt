현재 미로찾기 화면을 첨부한 참고 이미지와 비슷한 구조로 수정해 줘.
참고 이미지는 디자인과 화면 배치 참고용이며, 기존 웹앱의 기능을 삭제하거나 새 프로젝트로 다시 만들지 말고 현재 프로젝트를 수정해 줘.

1. 활동 전환 기능 유지 및 복구

화면 왼쪽 위에 다음 탭을 항상 표시해 줘.

미로찾기
라인트레이싱

현재 사라진 라인트레이싱 기능을 반드시 복구해 줘.
미로찾기 화면을 수정하면서 라인트레이싱 관련 코드, 버튼, 상태값, 화면을 삭제하지 마.

활동을 전환해도 선택한 난이도는 유지되게 해 줘.

2. 미로찾기 명령 블록

다음 네 종류의 명령 블록을 제공해 줘.

앞으로 이동
뒤로 이동
왼쪽 돌기
오른쪽 돌기

학생이 블록 버튼을 누르면 명령 순서 영역에 새 블록이 추가되게 해 줘.

각 명령 블록을 클릭하면 해당 블록의 설정 패널이 열리게 해 줘.
모든 명령을 한꺼번에 수정하는 방식이 아니라, 선택한 명령 블록 하나의 값만 수정하게 해 줘.

3. 회전 각도 미세 조정

왼쪽 돌기와 오른쪽 돌기에는 회전 각도를 학생이 직접 조절하는 기능을 넣어 줘.

각도 범위:

최소 1도
최대 180도
기본값 90도
소수점 첫째 자리까지 입력 가능

각도 조정 방법을 세 가지 모두 제공해 줘.

각도 슬라이더
숫자 직접 입력창
미세 조정 버튼

미세 조정 버튼:

−10°
−1°
−0.1°
+0.1°
+1°
+10°

숫자 입력창에서 87.3처럼 직접 입력할 수 있게 해 줘.
입력값이 슬라이더 범위에서 벗어나면 1도 또는 180도로 자동 보정해 줘.

명령 순서에 표시되는 블록에도 설정값을 표시해 줘.

예:

왼쪽 돌기 87.3°
오른쪽 돌기 92.6°

각도를 변경하면 화면 속 햄스터가 실행 시 정확히 해당 각도만큼 회전하게 해 줘.
회전 방향과 햄스터가 바라보는 방향을 화살표로 분명히 보여 줘.

4. 이동 거리 미세 조정

앞으로 이동과 뒤로 이동에는 다음 설정을 넣어 줘.

속도: 0~100
이동 시간: 0.1~5초
시간은 0.1초 단위로 조절
숫자 직접 입력 가능

학생이 실제 햄스터 실습처럼 값을 조금씩 수정해 볼 수 있게 다음 버튼도 넣어 줘.

시간 −0.1초
시간 +0.1초
속도 −1
속도 +1

시뮬레이션에서 이동 거리는 속도와 시간에 비례하게 계산해 줘.

명령 블록에는 다음처럼 표시해 줘.

앞으로 / 속도 50 / 1.5초
뒤로 / 속도 30 / 0.8초
5. 명령 편집과 실행

명령 순서 영역에서는 다음 기능이 작동해야 해.

드래그하여 순서 변경
개별 블록 삭제
마지막 명령 삭제
전체 명령 삭제
한 단계 실행
전체 실행
일시정지
다시 시작

다시 시작을 누르면 햄스터의 위치와 방향만 출발 상태로 돌아가고, 학생이 작성한 명령 목록은 유지해 줘.

별도로 명령 모두 지우기 버튼을 눌렀을 때만 명령을 삭제해 줘.

벽에 닿으면 실행을 멈추고 충돌한 위치를 표시해 줘.
도착점에 닿으면 성공 메시지를 표시해 줘.

6. 난이도별 도움 수준
쉬움
필요한 명령 순서는 일부 제공
학생은 속도, 시간, 각도 값을 수정
시작 방향을 크게 표시
목표까지 예상 경로를 흐린 점선으로 표시할 수 있는 힌트 버튼 제공
보통
첫 번째 명령만 제공
나머지는 학생이 추가
예상 경로 힌트는 한 번만 사용 가능
어려움
명령을 전혀 제공하지 않음
학생이 명령과 값을 모두 직접 구성
예상 경로 힌트 없음

새 미션을 만들 때마다 출발점, 출발 방향, 목표점, 장애물 위치가 달라지게 해 줘.

7. 라인트레이싱 기능 복구

라인트레이싱 탭을 선택하면 미로찾기와 완전히 다른 조작 화면을 보여 줘.

라인트레이싱 화면에는 다음 기능을 넣어 줘.

흰 바닥 위의 검은색 주행선
화면 속 햄스터 로봇
왼쪽 바닥 센서 상태
오른쪽 바닥 센서 상태
좌우 센서값 실시간 표시
기준값 조절
기본 속도 조절
방향 보정 세기 조절
실행
일시정지
처음부터 다시
새 코스

학생이 직접 조절할 값:

기본 속도 0~100
왼쪽 보정값 0~100
오른쪽 보정값 0~100
검은색과 흰색을 구분하는 센서 기준값

실행 중에는 왼쪽 센서와 오른쪽 센서가 검은 선 또는 흰 바닥 중 무엇을 감지하는지 눈에 보이게 표시해 줘.

센서 상태에 따라 다음 동작이 일어나게 해 줘.

두 센서가 모두 흰색이면 앞으로 이동
왼쪽 센서가 검은색이면 한쪽 방향으로 보정
오른쪽 센서가 검은색이면 반대 방향으로 보정
두 센서가 모두 검은색이면 정지 또는 교차로 처리

단, 센서와 보정 방향의 관계는 화면에 하드코딩된 설명으로만 보여 주지 말고, 실제 햄스터의 움직임을 보며 학생이 이해할 수 있게 해 줘.

코스 종류:

직선
완만한 곡선
S자
급커브
교차로 포함 코스

난이도에 따라 선의 곡률과 코스 복잡도를 다르게 해 줘.

8. 화면 배치

첨부한 참고 이미지처럼 데스크톱에서는 다음과 같이 배치해 줘.

왼쪽: 미로 또는 라인트레이싱 코스
오른쪽 위: 실행 버튼
오른쪽 가운데: 명령 또는 센서 설정
오른쪽 아래: 작성한 명령 순서
위쪽: 미로찾기/라인트레이싱 탭과 난이도 선택

아이패드 세로 화면에서는:

코스 화면
실행 버튼
설정 패널
명령 순서

순으로 세로 배치해 줘.

버튼과 슬라이더는 손가락으로 조작하기 충분히 크게 만들어 줘.

기존 미로찾기 기능을 새로 단순화하지 말 것
라인트레이싱 기능을 삭제하지 말 것
인쇄용 학습지나 기록표를 다시 만들지 말 것

각도는 90도 고정이 아니라 0.1도 단위까지 미세 조정 가능하게 할 것
실제 코드 수정 후 미로찾기와 라인트레이싱 두 화면을 모두 직접 테스트할 것
오류가 있으면 수정한 뒤 완료했다고 알려 줄 것
